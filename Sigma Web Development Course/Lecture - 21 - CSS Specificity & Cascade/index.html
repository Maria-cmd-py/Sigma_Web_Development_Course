CSS Specificity

What is CSS Specificity?

Jab multiple classes apply hoon, to last wali CSS rule (.cpurple { color: purple; }) apply hoti hai, kyunki sab class selectors ki specificity same hoti hai, aur jo baad mein likha ho wo jeet jata hai.

Reason:

All three classes .yellow, .cred, and .cpurple have same specificity (class = 0,1,0). Last defined rule in CSS wins â†’ So, purple color apply hoga.

Priorities:

  • Inline styles (highest priority)
  • ID selectors
  • Class selectors
  • Element selectors (lowest priority)

Why in Specificity Priority Matters?

Specificity priority ka matlab hai ki agar multiple CSS rules apply ho rahe hain, to browser ko pata hota hai ki kaunsa rule zyada specific hai aur usi ko apply karta hai. Isse developers ko control milta hai ki unki styles kaise render hongi.

Example:

Suppose agar h1 tag mein inline style add karte hain, to wo sabse zyada specific hoga aur usi ka color apply hoga.

Inline Style Example

Yahan h1 tag ka color green hai kyunki inline style ka specificity highest hota hai.

Cascade Algorithm:

Cascade algorithm ka matlab hai ki agar multiple CSS rules apply ho rahe hain, to browser unhe ek order mein evaluate karta hai. Is order ko samajhna zaroori hai taaki aap apne styles ko sahi tarike se apply kar sakein.

Specificity Calculation:

Specificity calculate karne ke liye, aapko CSS selectors ki specificity value samajhni hogi. Ye value different types of selectors ke liye alag hoti hai. Jaise:

  • Inline styles: 1,0,0,0
  • ID selectors: 0,1,0,0
  • Class selectors: 0,0,1,0
  • Element selectors: 0,0,0,1
